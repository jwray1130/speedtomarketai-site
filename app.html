Settings

Model and token settings for the pipeline. The Anthropic API key is held server-side — you do not need to paste it here.

Default global model. Individual modules may override this based on reasoning / extraction preference.
Off (default): pipeline uses per-module routing — most modules run on Opus, three lighter ones run on Sonnet, classifier verification on Sonnet. On: every single LLM call (classifier, verifier, all 17 modules) goes through the model above. Use this for max-quality runs regardless of cost. ~50% more spend per submission with all-Opus vs. default routing.
Anthropic only generates as much as the response naturally needs — this is the ceiling, not the target. Default 4096 covers most modules. Raise to 8192 or 16384 for complex Loss History tables, large Excess Towers, or detailed Discrepancy reports. Server-side limit: 16384.
